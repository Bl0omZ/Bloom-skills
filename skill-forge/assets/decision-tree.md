# 模式选择决策树

根据 Phase 1 采访收集的三个答案，按以下规则树选择主模式。

## 规则树

```
任务描述中包含「注入知识/文档/规范/API/框架」？
├── YES → Tool Wrapper
└── NO ↓

任务描述中包含「生成/创建/写/产出/模板」？
├── YES →
│   触发条件是「用户提供结构化输入」？
│   ├── YES → Generator
│   └── NO（需要先收集需求）→ Inversion + Generator 组合
└── NO ↓

任务描述中包含「检查/审查/评分/验证/质量」？
├── YES → Reviewer
└── NO ↓

任务描述中包含「流程/步骤/顺序/pipeline/工作流」？
├── YES → Pipeline
└── NO ↓

任务需要在执行前收集用户信息？
├── YES → Inversion
└── NO → Tool Wrapper（默认：注入上下文知识）
```

## 评分细化（多模式竞争时使用）

对每个候选模式打分（0-3分），选得分最高者：

| 信号词 | Tool Wrapper | Generator | Reviewer | Inversion | Pipeline |
|--------|-------------|-----------|----------|-----------|----------|
| 注入/文档/规范/API | +3 | 0 | 0 | 0 | 0 |
| 生成/创建/写/模板 | 0 | +3 | 0 | +1 | +1 |
| 检查/审查/评分/分级 | 0 | 0 | +3 | 0 | +1 |
| 步骤/顺序/流程/工作流 | 0 | +1 | +1 | 0 | +3 |
| 采访/收集/问/了解需求 | 0 | +1 | 0 | +3 | 0 |
| 每次/自动/定时/触发 | +1 | +2 | +1 | 0 | +2 |
| 外部用户/最终用户受众 | +1 | +1 | 0 | +2 | +1 |
| 结构化输出/JSON/YAML | 0 | +2 | +1 | 0 | +2 |

**同分处理：** 得分相同时，优先级顺序：Inversion > Generator > Reviewer > Pipeline > Tool Wrapper
（Inversion 优先，因为信息不足时采访比猜测更安全）

## 受众影响

| 受众 | 偏向模式 |
|------|----------|
| 开发者 | Tool Wrapper、Reviewer |
| 最终用户 | Inversion、Generator |
| 系统/下游 agent | Generator（结构化输出）、Pipeline |
